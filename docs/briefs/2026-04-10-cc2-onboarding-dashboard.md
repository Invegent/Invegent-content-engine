Read `docs/00_sync_state.md` before starting.

### 1. SECURITY DEFINER functions needed (apply as migrations)

**a) Get all submissions with package details**
```sql
CREATE OR REPLACE FUNCTION public.get_onboarding_submissions(
  p_status TEXT DEFAULT NULL
) RETURNS TABLE (
  submission_id UUID, status TEXT, contact_name TEXT, contact_email TEXT,
  business_name TEXT, industry_vertical TEXT, selected_package_code TEXT,
  package_label TEXT, base_price_aud NUMERIC,
  desired_platforms TEXT[], submitted_at TIMESTAMPTZ,
  info_request_count INTEGER, reviewed_at TIMESTAMPTZ
) LANGUAGE plpgsql SECURITY DEFINER AS $$
BEGIN
  RETURN QUERY
  SELECT s.submission_id, s.status, s.contact_name, s.contact_email,
         s.business_name, s.industry_vertical, s.selected_package_code,
         sp.label, sp.base_price_aud,
         s.desired_platforms, s.submitted_at,
         s.info_request_count, s.reviewed_at
  FROM c.onboarding_submission s
  LEFT JOIN c.service_package sp
    ON sp.package_code = s.selected_package_code AND sp.is_current = true
  WHERE (p_status IS NULL OR s.status = p_status)
  ORDER BY s.submitted_at DESC;
END;
$$;
GRANT EXECUTE ON FUNCTION public.get_onboarding_submissions(TEXT) TO service_role;
```

**d) Approve submission — the critical function**
```sql
CREATE OR REPLACE FUNCTION public.approve_onboarding(
  p_submission_id UUID,
  p_operator_email TEXT
) RETURNS TABLE (client_id UUID, portal_user_id UUID)
LANGUAGE plpgsql SECURITY DEFINER AS $$
DECLARE
  v_sub     c.onboarding_submission%ROWTYPE;
  v_pkg     c.service_package%ROWTYPE;
  v_client  UUID;
  v_pu_id   UUID;
BEGIN
  -- Load submission
  SELECT * INTO v_sub FROM c.onboarding_submission WHERE submission_id = p_submission_id;
  IF NOT FOUND THEN RAISE EXCEPTION 'Submission not found'; END IF;
  IF v_sub.status = 'approved' THEN RAISE EXCEPTION 'Already approved'; END IF;

  -- Load package
  SELECT * INTO v_pkg FROM c.service_package
  WHERE package_code = v_sub.selected_package_code AND is_current = true;

  -- Create client
  INSERT INTO c.client (client_name, slug, status, timezone, contact_email)
  VALUES (
    v_sub.business_name,
    lower(regexp_replace(v_sub.business_name, '[^a-zA-Z0-9]+', '-', 'g')),
    'active',
    'Australia/Sydney',
    v_sub.contact_email
  ) RETURNING client_id INTO v_client;

  -- Create portal_user entry
  INSERT INTO portal_user (client_id, email)
  VALUES (v_client, v_sub.contact_email)
  RETURNING portal_user_id INTO v_pu_id;

  -- Create client_service_agreement
  INSERT INTO c.client_service_agreement (
    client_id, submission_id, service_package_id,
    monthly_price_aud, billing_email, billing_contact_name,
    payment_method, payment_terms_days,
    agreement_template_version, signed_by_name, signed_by_ip, signed_at,
    effective_from, status
  ) VALUES (
    v_client, p_submission_id, v_pkg.service_package_id,
    v_pkg.base_price_aud, v_sub.contact_email, v_sub.contact_name,
    'invoice', 14,
    v_sub.agreement_version, v_sub.agreement_signed_name,
    v_sub.agreement_signed_ip, v_sub.agreement_accepted_at,
    CURRENT_DATE, 'active'
  );

  -- Mark submission approved
  UPDATE c.onboarding_submission SET
    status = 'approved',
    reviewed_by = p_operator_email,
    reviewed_at = NOW(),
    updated_at = NOW()
  WHERE submission_id = p_submission_id;

  RETURN QUERY SELECT v_client, v_pu_id;
END;
$$;
GRANT EXECUTE ON FUNCTION public.approve_onboarding(UUID, TEXT) TO service_role;
```

### 2. Dashboard Actions — `actions/onboarding.ts` (new file in invegent-dashboard)

```typescript
'use server'

// getSubmissions(status?: string) — list all submissions
// getSubmissionDetail(submissionId: string) — full detail for review
// requestMoreInfo(submissionId, missingFields, notes) — flags and emails client
// approveSubmission(submissionId) — calls approve_onboarding(), then triggers magic link
// rejectSubmission(submissionId, reason) — marks rejected
```

**approveSubmission() must:**
1. Call `approve_onboarding()` DB function → gets client_id back
2. Call Supabase Admin API to invite user by email (sends magic link to portal.invegent.com)
   ```typescript
   const { data, error } = await supabaseAdmin.auth.admin.inviteUserByEmail(
     submission.contact_email,
     { redirectTo: 'https://portal.invegent.com/callback' }
   )
   ```
3. Call `onboarding-notifier` Edge Function with `{ type: 'approved', submission_id }` to send the welcome email
4. Return success

**requestMoreInfo() must:**
1. Generate a UUID update_token
2. Call `request_onboarding_info()` DB function
3. Call `onboarding-notifier` Edge Function with `{ type: 'needs_info', submission_id }` to send the request email
   - The email includes a link: `portal.invegent.com/onboard/update?id={submission_id}&token={update_token}`
4. Return success

### 8. Portal — Update Submission Route (minimal, for CC2)

Create `app/onboard/update/page.tsx` in `invegent-portal`.

This is the page clients land on when they click "update your application" from the needs_info email.

**Behaviour:**
- Read `?id=` and `?token=` from URL params
- Validate: fetch submission by id, check token matches `update_token` column
- If invalid: show "This link has expired or is invalid. Email hello@invegent.com."
- If valid: show a simplified form with ONLY the flagged fields (from `missing_fields` JSONB)
  - Each field shows the field label + reason from the operator
  - Client fills in only those fields
  - Submit → calls `public.update_onboarding_submission()` DB function (see below)
  - Success screen: "Thank you — we'll review your update within 1 business day."

**DB function needed:**
```sql
CREATE OR REPLACE FUNCTION public.update_onboarding_submission(
  p_submission_id UUID,
  p_token UUID,
  p_updates JSONB
) RETURNS void LANGUAGE plpgsql SECURITY DEFINER AS $$
BEGIN
  -- Validate token
  IF NOT EXISTS (
    SELECT 1 FROM c.onboarding_submission
    WHERE submission_id = p_submission_id AND update_token = p_token
  ) THEN
    RAISE EXCEPTION 'Invalid or expired update token';
  END IF;

  -- Apply updates dynamically (only fields present in p_updates)
  UPDATE c.onboarding_submission SET
    abn              = COALESCE(p_updates->>'abn', abn),
    facebook_page_url = COALESCE(p_updates->>'facebook_page_url', facebook_page_url),
    instagram_url    = COALESCE(p_updates->>'instagram_url', instagram_url),
    linkedin_url     = COALESCE(p_updates->>'linkedin_url', linkedin_url),
    website_url      = COALESCE(p_updates->>'website_url', website_url),
    contact_phone    = COALESCE(p_updates->>'contact_phone', contact_phone),
    industry_detail  = COALESCE(p_updates->>'industry_detail', industry_detail),
    -- add other updatable fields as needed
    status           = 'pending',  -- back to pending for operator review
    missing_fields   = NULL,
    update_token     = NULL,       -- invalidate token after use
    updated_at       = NOW()
  WHERE submission_id = p_submission_id AND update_token = p_token;
END;
$$;
GRANT EXECUTE ON FUNCTION public.update_onboarding_submission(UUID, UUID, JSONB) TO anon;
```
