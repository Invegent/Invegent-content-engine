## 🔴 NEW FINDING — F-YT-PUB-PUBLISH-AUDIT-GAP (open, candidate brief)

**Cross-posted YouTube publishes are marked `published` with a real `youtube_video_id` but get NO `m.post_publish` youtube audit row.**

- **Root cause:** `m.post_publish` has `uq_publish_attempt UNIQUE (post_draft_id, attempt_no)` (NOT per-platform). The publisher's success-path INSERT hardcodes `attempt_no: 1` (index.ts:255). A cross-posted draft already has a Facebook `post_publish` row with `attempt_no=1`, so the YouTube INSERT collides on `(post_draft_id, attempt_no)` → rejected. The publisher logs `insertErr` but does **not** throw (index.ts:258), so `video_status='published'` is committed while the YouTube audit row is silently dropped.
- **Confirmed:** the 2 published B2 drafts have only their April Facebook `post_publish` row, no youtube row; only **1 draft in all history** has both FB+YT publish rows.
- **Scope:** pre-existing latent bug (hardcoded `attempt_no=1` since v1.5.0) — but v1.9.0's predicate broadening is precisely what now routes cross-posted (`published`) drafts to YouTube, i.e. exactly the colliding class. **All 17 B2 drafts are cross-posted (FB)** → all 17 will hit this as they drain.
- **What is NOT broken:** the uploads succeed (videos are live on YouTube); no double-upload risk (the SELECT requires `video_status='generated'`, which flips to `published` on success, so a draft is never re-picked); the gap is backfillable from `draft_format.youtube_video_id`.
- **Impact:** the `m.post_publish` YouTube audit trail is incomplete for cross-posted drafts (affects dashboard/insights/reconciliation that read `post_publish`).

**Recommended follow-on (candidate brief, gated):**
1. **Publisher fix (v1.10.0):** make the YouTube `post_publish` INSERT use a non-colliding `attempt_no` (e.g. `max(attempt_no)+1` for the draft) or change the INSERT to be platform-distinct; consider whether the uniqueness should be `(post_draft_id, platform, attempt_no)`. Also: stop swallowing `insertErr` silently (surface it, since a published-without-audit-row is a real gap).
2. **Backfill:** insert the missing YouTube `post_publish` rows for the affected cross-posted drafts (the 2 now + the rest as they drain) from `draft_format.youtube_video_id`, with a non-colliding `attempt_no` — `sql_destructive`-gated.
