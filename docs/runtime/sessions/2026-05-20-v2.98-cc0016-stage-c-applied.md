## Attachment shape validation (live)

Validation is enforced in two concordant places. The pipeline-layer check in `emit_event` enforces only array-shape + max-3 invariants so direct emitters (cron, triggers) fail fast on malformed shape. The full per-item check (storage_path + mime_type + MIME whitelist) lives in `fn_emit_manual_event` since manual FAB events are the only path that legitimately submits attachments.

| Layer | Invariant | Error message |
|---|---|---|
| `emit_event` (pipeline) | `jsonb_typeof(p_attachments) = 'array'` | `emit_event: p_attachments must be a JSON array` |
| `emit_event` (pipeline) | `jsonb_array_length(p_attachments) <= 3` | `emit_event: attachments limited to 3 per event` |
| `fn_emit_manual_event` (manual) | array type | `attachments must be a JSON array` |
| `fn_emit_manual_event` (manual) | length ≤ 3 | `attachments limited to 3 per event` |
| `fn_emit_manual_event` (manual) | each item has non-null `storage_path` + `mime_type` | `each attachment must include storage_path and mime_type` |
| `fn_emit_manual_event` (manual) | mime_type in (`image/jpeg`, `image/png`, `image/webp`) | `attachment mime_type X not allowed` |

Storage `file_size_limit=5242880` (5 MB per file) and bucket-level MIME whitelist (Stage A) remain enforced at the Supabase Storage layer on upload (defense-in-depth).

## L-v2.98-X candidates

**L-v2.98-a candidate (MEDIUM-SIGNAL, watcher)**: when extending an existing SECURITY DEFINER function's argument list with `DEFAULT`-bearing trailing parameters, **DROP the prior overload before CREATE OR REPLACE** to prevent silent overload coexistence. PostgreSQL treats different `proargnames` lengths as distinct functions, not as REPLACE targets. The brief v1.1 patch made this explicit (Step 1 + Step 2 DROP before Step 3 + Step 4 CREATE). Tag for re-exercise next time a function signature extends.

**L-v2.98-b candidate (MEDIUM-SIGNAL, methodology)**: when a Type-C echo pushback returns from D-01 review on a patched brief, classify each pushback as either (i) genuinely new concern (resolve in brief), or (ii) Type-C echo of structural risk already addressed by the patched design (cite the design element + V-check that proves resolution in the next D-01 fire's corrected_action). Avoid re-patching the brief in response to (ii) — that risks introducing drift. The dbabb576 fire's 3 pushbacks were correctly classified as Type-C echoes already resolved by DROP+CREATE atomicity + delegation-not-INSERT contract + explicit per-item schema in brief v1.1 lines 110/580-592/672-673.

**L-v2.98-c candidate (minor, tooling)**: Supabase MCP `execute_sql` runs as `postgres` superuser by default. To test grants for non-superuser roles, use `SET LOCAL ROLE <role>` within the same transaction. Temp tables created by the superuser are NOT writable by the role-switched session — pass results via `SELECT` return rather than temp-table INSERT.
