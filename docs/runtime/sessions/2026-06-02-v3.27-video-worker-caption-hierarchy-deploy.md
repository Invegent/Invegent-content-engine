## Lesson

- **L-v3.27-a:** `./scripts/safe-deploy.sh` gates on the **cached** `m.vw_ef_drift_current` register (not a live recompute), so the mandatory deploy path is **merge → targeted EF-drift refresh → B-FD → `--allow-warn` deploy → final refresh → A-LE**; never bypass the wrapper. New wrinkle this lane: `drift-check` reads the repo side from `raw.githubusercontent.com/.../main/...`, which can lag (and return inconsistently across CDN nodes) for a few minutes after a push — so a refresh immediately post-push may mis-report `A-LE`. **Poll the targeted refresh until the persisted register shows `B-FD` before running the deploy.** (Reinforces L-v3.25-a / L-v3.26-a.)
