### L34 — `k.fn_sync_registry` auto-registration is part of database architecture

The function `k.fn_sync_registry` (called by `k.evtrg_sync_registry_on_create_table`) is invariantly active for all `CREATE TABLE` in registered schemas. It auto-INSERTs 1 stub `k.table_registry` row + N stub `k.column_registry` rows. Brief authoring strategy must accommodate this — not work around it.

### L35 — `INSERT ... ON CONFLICT DO UPDATE` is the defensive pattern for k.* registry rows

Path B (ON CONFLICT) survives whether trigger fires first, is disabled, or has different semantics. v5 §3.3 + §3.4 use ON CONFLICT for both `k.table_registry` (conflict target `(schema_name, table_name)`) and `k.column_registry` (conflict target `(table_id, column_name)`).
