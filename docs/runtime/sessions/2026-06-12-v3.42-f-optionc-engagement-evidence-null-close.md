## 1. Lane summary

`m.post_format_performance.avg_engagement_rate` is effectively NULL — despite `post_count` accumulating — because of **three stacked defects**:

**D1 — Raw FB engagement acquisition failure (primary).**
- The direct Graph metric `post_engaged_users` **fails 526/526 FB rows** across all four clients/tokens (`raw_payload.failed_metrics` proves it; systematic, not token-specific; introduced as a "direct metric" in insights-worker v14.0.0).
- `engaged_users` is therefore **NULL 526/526**.
- The derived fallback basis (reactions+comments+shares) is **0 on 526/526 rows** — and the code only uses the derivation when > 0; failures in the post-fields call are coerced to 0, so "fetch failed" and "genuinely zero engagement" are indistinguishable in the stored data.
- Therefore `engagement_rate` is **NULL for all FB rows even where reach exists** (reach > 0 on 322/526; reach collection mostly healthy — only 67 failed-reach rows, all NDIS-Yarns).

**D2 — Aggregation dual-writer clobber.**
- insights-worker's own `computeFormatPerformance` (runs per-client at ~03:01–03:16 UTC after each FB insights cron; windows **30+90**; **all platforms, no reach filter**) writes via `upsert_format_performance` and *does* capture the YouTube engagement values.
- `refresh_post_format_performance()` (cron `refresh-format-performance-daily` 03:15; windows **7/30/0**; **FB-only** via its `perf.reach IS NOT NULL` filter) writes the **same table on the same PK** (client, format, window).
- The 03:15 refresh **clobbers useful worker-produced 30-day values with NULL FB-only averages** for every combo that has ≥1 FB perf row.
- **The Advisor reads exactly the damaged window-30 slice.** Survivor proof: every non-NULL `avg_engagement_rate` row is window-90 (refresh never writes 90) or a YT-only video combo outside refresh's FB set — all `computed_at` 03:01 (NY) / 03:12 (PP), i.e. fresh worker output that escaped the 03:15 clobber. Example: PP × video_short_kinetic same day — window-90 (worker) `post_count=36, avg=0.0000`; window-30 (refresh-clobbered) `post_count=15, avg=NULL`.

**D3 — YouTube structurally excluded.**
- YT perf rows have engagement values on some rows (20/62 non-null) but **`reach` is NULL 62/62**.
- The refresh filter requires `reach IS NOT NULL` → **every YT row is excluded** from `post_format_performance` via the refresh path.
- YT insights are also **stale (latest 05-29) / manual-only** — existing carry. (Correction to prior framing: FB insights ARE cron-scheduled — per-client jobs ny 03:00 / cfw 03:05 / pp 03:10 / invegent 03:15 + `insights-feedback-daily` 03:30; the FB schedule layer is healthy.)

**Also recorded:**
- **LinkedIn and Instagram have NO insight rows at all** (no LI/IG insights collection exists).
- **Advisor receives window-30 rows where NULL is coerced to `0`** (ai-worker `8204a5c7`: `Number(p.avg_engagement_rate ?? 0)`) → the Advisor is injected with **"0.0% avg engagement" per format on every job — misleading rather than merely absent** (a fabricated all-zero performance summary).
- **Option C remains PARKED because the evidence feed is structurally broken** — this lane confirms Option C **cannot become evidence-based until the engagement feed is repaired**; no amount of waiting/accumulation fixes it.
